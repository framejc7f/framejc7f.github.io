# Финансовый помощник

Небольшое фронтенд-приложение для личного учета финансов: доходы, расходы, обязательные платежи, возможный доход, календарь и аналитика.

Сайт рассчитан на запуск как статический проект, в том числе через GitHub Pages.

## Возможности

- Учет доходов и расходов
- История операций с фильтрами
- Обязательные платежи
- Регулярные платежи:
  ежедневные, еженедельные, ежемесячные
- Возможный доход с подтверждением
- Календарь операций и платежей
- Базовая аналитика по расходам и динамике
- Импорт и экспорт данных в JSON
- Хранение данных в `localStorage`
- Адаптация под мобильные устройства


## Как запустить локально

Важно: проект использует `ES modules`, поэтому не стоит открывать `index.html` двойным кликом как `file://...`. В этом режиме браузер может не загрузить JavaScript корректно.

### Вариант 1. Через Python

Открой терминал в папке проекта и выполни:

```powershell
py -m http.server 8000
```

Если `py` не работает:

```powershell
python -m http.server 8000
```

После этого открой в браузере:

[http://localhost:8000](http://localhost:8000)

## Деплой на GitHub Pages

1. Загрузить проект в GitHub-репозиторий
2. Убедиться, что в репозиторий попали `index.html` и папка `src`
3. В настройках репозитория открыть `Settings -> Pages`
4. Выбрать ветку для публикации, например `main`
5. Сохранить настройки

После публикации сайт будет доступен по адресу вида:

`https://username.github.io/repository-name/`

## Где хранятся данные

Все данные сохраняются в браузере пользователя через `localStorage`.

Основные ключи:

- `finance_real_transactions`
- `finance_obligations`
- `finance_potential_incomes`
- `finance_obligations_collapsed`
- `finance_calendar_collapsed`
- `finance_history_collapsed`

Если очистить данные сайта в браузере, история и настройки будут удалены.

## Основные сценарии использования

### Обычные операции

- Добавление доходов и расходов
- Просмотр истории
- Редактирование операции
- Удаление операции

### Обязательные платежи

- Добавление разового платежа
- Добавление регулярного платежа
- Оплата платежа с переносом на следующую дату для регулярных
- Редактирование платежа
- Удаление платежа

### Возможный доход

- Добавление записи на дату в календаре
- Редактирование
- Удаление
- Подтверждение с переводом в реальный доход

## Ограничения

- Данные хранятся только локально в браузере
- Нет серверной части и авторизации
